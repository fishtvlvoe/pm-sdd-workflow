# 輸入處理說明

這份文件只放操作細節,不放 Prompt 內容本身。Prompt 內容在 `docs/02-AI協作提示詞-需求釐清四步驟.md`。

## 判斷輸入是檔案路徑還是貼上的文字

- `/需求轉譯` 後面的內容看起來像路徑(例如包含 `docs/`、`.md`、`.txt`、`.docx` 副檔名,或以 `/` 開頭、或明顯是相對路徑)→ 當作檔案路徑,直接讀取。
- 內容是一大段話、沒有明顯路徑格式 → 當作直接貼上的逐字稿/筆記文字,不要嘗試當路徑去讀。
- 兩者都沒附(指令後面是空的)→ 停下來問使用者要分析哪份資料,不要自己亂猜檔案(例如不要自動抓 `docs/` 底下看起來最新的檔案)。

## 找不到檔案時怎麼辦

- 讀取後發現檔案不存在 → 用 `ls` 確認附近路徑有沒有打錯字或副檔名不同(例如客戶原始檔可能是 `.docx` 不是 `.md`),抓到相近檔名就跟使用者確認是不是這一份,不要沉默地自己選一份代替。
- 確認過路徑仍找不到 → 停止,告知使用者找不到這個檔案,不要憑記憶編內容頂替。

## 多個步驟之間資料怎麼傳遞

- 每個步驟的輸出(待確認清單/需求分類表/雛形功能分析結果)留在當前對話裡,不主動寫成檔案,下一步驟直接沿用同一個對話裡剛產出的內容當輸入,不用重新讀取。
- 如果使用者中途手動調整了某個步驟的輸出(例如改了待確認清單的某一項),用使用者調整後的版本繼續往下,不要用調整前的舊版本。
- 步驟 3(雛形功能分析)如果使用者沒提供雛形/Demo 資料,直接跳過,步驟 4 的輸入就少這一份,不用假裝有雛形資料硬湊。
- 步驟 4 完成後,若使用者要求存檔,才問清楚要存哪個路徑;預設不主動寫檔案。
